# F1 2024 Season Globe ŌĆö Design Spec

**Date:** 2026-04-07  
**Status:** Approved

## Overview

Transform the existing three-globe Next.js app into an interactive 2024 F1 season globe. The globe displays all 24 circuits as glowing markers, connected by sequential animated arcs showing the F1 circus's journey around the world. Hovering a circuit shows a tooltip with race details.

---

## Data & Architecture

### New file: `lib/data/f1-2024.js`

Exports an array of 24 race objects, one per round:

```js
{
  round: 1,
  name: "Bahrain Grand Prix",
  location: "Sakhir",
  country: "Bahrain",
  date: "2 March 2024",
  lat: 26.0325,
  lng: 50.5106,
}
```

From this array, arcs are generated by pairing consecutive races (round N ŌåÆ round N+1). Arc altitude scales with geographic distance: short regional hops arc low (~0.15), intercontinental jumps arc high (~0.6).

### Fix: missing `World` export in `lib/components/ui/globe.jsx`

The `World` component is imported by both `pages/index.js` and `lib/components/globe.jsx` but is not currently exported. It must be added ŌĆö it is the Canvas wrapper that mounts `GlobeComponent`, scene lights, and `OrbitControls`.

### `pages/index.js`

Becomes the F1 page. Imports F1 data, generates arcs, manages tooltip state, renders the fullscreen globe with title overlay.

The existing `lib/components/globe.jsx` (GlobeDemo) is left as-is but not used by the main page.

## Globe Visuals & Animation

### Globe config

| Property | Value | Notes |
|---|---|---|
| `globeColor` | `#0a0a0a` | Near-black |
| `emissive` | `#1a0000` | Faint red underglow |
| `atmosphereColor` | `#ff1801` | F1 red halo |
| `atmosphereAltitude` | `0.15` | Cinematic broadcast look |
| `polygonColor` | `rgba(255,255,255,0.15)` | Subtle country outlines |
| `autoRotate` | `true` | |
| `autoRotateSpeed` | `0.3` | Slow, cinematic |
| `initialPosition` | `{ lat: 48, lng: 10 }` | Centered on Europe |

### Arc config

- Color: F1 red `#e8002d`
- Stroke: `0.4`
- `arcDashInitialGap` based on round number so arcs animate in calendar sequence
- Altitude: computed per arc based on haversine distance between consecutive circuits

### Circuit markers

- `pointsData`: all 24 circuit lat/lng positions
- `pointRadius`: `2.5`
- `pointColor`: `#e8002d`
- `pointAltitude`: `0.01`

### Ring pulses

Ring propagation fires on circuit points on a 2-second interval (existing mechanic), restricted to circuit points rather than random arc endpoints.

## Tooltip & UI

### Page layout

- Fullscreen (`h-screen`), black background
- Globe fills the screen; no container constraints

### Header overlay (top-center, framer-motion fade-in)

- Title: **"2024 FORMULA 1 SEASON"** ŌĆö bold condensed caps, white
- Subtitle: **"24 Grands Prix ┬Ę 21 Countries"** ŌĆö muted red (`#e8002d` at 80% opacity)

### Tooltip card

Triggered by `onPointHover` callback on the globe's point layer. The callback receives `(point, event)` ŌĆö `event.clientX / clientY` provides screen coordinates for positioning. Card is a React-rendered absolute-positioned overlay at those coordinates.

**Content:**
- Round badge: e.g. "RD 01" ŌĆö red background, white text
- Race name: e.g. "Bahrain Grand Prix" ŌĆö white, bold
- Location + country: e.g. "Sakhir, Bahrain" ŌĆö muted
- Date: e.g. "2 March 2024" ŌĆö muted

**Styling:** dark card (`#1a1a1a` bg), subtle border (`#333`), disappears on mouse-out. Globe wrapper has `pointer-events-none` removed only on point hit areas (handled by three-globe's event system).
